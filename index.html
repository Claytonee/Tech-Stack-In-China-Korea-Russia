Kipengele: Multi-tenant Architecture

Multi-tenant systems zinapaswa kuzingatia tenant isolation, data partitioning, scalability na compliance (student data privacy). Hapa ni patterns 3 za kawaida:

  1. Shared DB, Shared Schema â€” cheap & simple, lakini haifai kwa isolation kali.
  2. Shared DB, Separate Schemas â€” balance ya cost vs isolation.
  3. Separate DB per tenant â€” isolation nzuri, gharama ya infra juu.

Security & Compliance

Kumbuka: education systems zinatakiwa kuhifadhi PII za wanafunzi. Muhimu:

  • Encryption at rest & in transit
  • Audit logs & role-based access control (RBAC)
  • Data residency (host in-country if law requires)
  • Regular pentesting & secure SDLC

Fast MVP Strategy for a Solo Dev

Starter DB Schema (simplified)

-- students, courses, enrollments
CREATE TABLE students (id UUID PRIMARY KEY, name TEXT, email TEXT UNIQUE, dob DATE, created_at TIMESTAMP);
CREATE TABLE courses (id UUID PRIMARY KEY, code TEXT, title TEXT, credits INT, created_at TIMESTAMP);
CREATE TABLE enrollments (id UUID PRIMARY KEY, student_id UUID REFERENCES students(id), course_id UUID REFERENCES courses(id), status TEXT, grade TEXT, created_at TIMESTAMP);
